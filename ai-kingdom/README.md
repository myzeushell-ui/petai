# AI Kingdom — «Ночь перед осадой»

Однопользовательская средневековая **командная стратегия**, где вы —
молодой правитель. Вы не двигаете юнитов мышью. Вы командуете **людьми** —
офицерами, которые командуют войсками, — отдавая приказы на естественном
русском языке (голосом или текстом).

> «Эдвард, возьми 300 копейщиков и удерживай мост до рассвета.»
> «Милорд, я исполню приказ, но без подкрепления мы не удержим мост долго…»

Это играбельный вертикальный срез на 10–20 минут: одна законченная партия
сценария **«Ночь перед осадой»** — от распределения войск до финального
решения о судьбе пленного вражеского командира.

---

## Быстрый старт

```bash
cd ai-kingdom
npm install
npm run dev        # запускает игру на http://localhost:5173
```

Другие команды:

```bash
npm run build      # проверка типов + production-сборка в dist/
npm run preview    # предпросмотр собранной версии
npm run test       # unit-тесты критической логики (vitest)
npm run typecheck  # строгая проверка типов
```

Требуется Node 18+. Внешних API, ключей и базы данных не нужно — игра
полностью работает локально.

> Проект живёт в подпапке `ai-kingdom/` внутри репозитория и не зависит от
> основного приложения PetAI.

## Поддержка голоса

| Функция | Технология | Браузеры |
|---|---|---|
| Голосовой **ввод** приказов | Web Speech API (`SpeechRecognition`) | Chrome, Edge (лучший русский), частично Safari |
| Голосовые **ответы** офицеров | `SpeechSynthesis` | Большинство современных браузеров |

Голос включается в **Настройках** (⚙ вверху). Если распознавание или синтез
недоступны — игра **полностью** работает через текст: распознанный текст
всегда показывается и его можно исправить перед отправкой.

## Как играть

1. **Выберите офицера** слева (совет офицеров).
2. **Отдайте приказ** справа — голосом (🎙) или текстом. Например:
   `Эдвард, удерживай мост до рассвета`.
3. **Проверьте разбор** приказа в карточке подтверждения и подтвердите его
   (для опасных приказов). Безопасные запросы исполняются сразу.
4. **Снимите паузу** вверху (или клавишей `Пробел`) — пойдёт время.
5. **Следите за докладами** на карте и в «Хронике ночи».
6. **Меняйте приказы** при необходимости — новый приказ офицеру заменяет
   старый. Офицеры **помнят** ваши решения.
7. Переживите ночь и **решите судьбу пленника**.

**Цель:** удержать замок до рассвета и разбить или отбросить врага. Ключ к
обороне — **Каменный мост**: узкий проход, где горстка людей задержит
превосходящую армию. Не оставляйте замок без гарнизона и берегите
продовольствие.

Горячие клавиши: `Пробел` — пауза/продолжить, `` ` `` — панель отладки,
`Esc` — закрыть окна.

## Какие приказы понимает прототип

Разбор локальный (словари + регулярные выражения + контекст) — не LLM.
Поддерживаются формулировки и синонимы, например:

| Действие | Примеры фраз |
|---|---|
| `MOVE` | «Эдвард, веди 200 копейщиков к мосту», «иди сюда» (после выбора локации) |
| `HOLD` / `DEFEND` | «удерживай мост до рассвета», «займи оборону на холмах» |
| `ATTACK` | «Роланд, атакуй фланг врага» |
| `SCOUT` | «Мара, разведай северный лес» |
| `AMBUSH` | «спрячь конницу за холмами и жди моего приказа» |
| `SUPPLY` | «перевези еду из деревни в замок» |
| `EVACUATE` | «эвакуируй жителей деревни» |
| `RETREAT` | «Эдвард, отступай к замку» |
| `PROTECT` / `REINFORCE` | «прикрой деревню», «пошли подкрепление к мосту» |
| `WAIT` | «жди моего приказа» |
| `ASK_STATUS` | «Мара, каковы наши запасы?», «Эдвард, сколько людей ты потерял?» |
| `ASK_ADVICE` | «что посоветуешь?» |
| `CANCEL_ORDER` | «отмени предыдущий приказ» |

Условия распознаются отдельно: `до рассвета`, `любой ценой`, `по моему приказу`,
`без потерь`, `скрытно`. Числа понимаются цифрами и словами («двести»,
«две сотни»). Если чего-то не хватает («отправляйся к мосту» без числа),
офицер **сам спросит** уточнение, а ваш следующий ответ («двести») будет
привязан к незаконченному приказу.

## Архитектура

Игровая логика — **чистые функции**, полностью отделённые от React. UI, голос
и «ИИ» никогда не меняют состояние напрямую: они лишь вызывают функции движка,
которые возвращают новое `GameState`.

```
src/
  game/                     ← вся симуляция (чистый TypeScript, тестируемый)
    types.ts                ← канонический контракт типов
    scenario.ts             ← КОНФИГУРАЦИЯ сценария (все числа сюда)
    commandInterpreter.ts   ← разбор естественного языка (CommandInterpreter)
    dialogue.ts             ← реплики офицеров (DialogueProvider)
    officers.ts             ← принятие приказов, инициатива, характеры
    orders.ts               ← жизненный цикл приказа, ETA, разбиение отрядов
    world.ts                ← поиск пути, движение, снабжение
    battle.ts               ← расчёт боя
    events.ts               ← сценарные события
    memory.ts               ← память офицеров
    prisoner.ts             ← финал с пленником
    engine.ts               ← оркестратор: createInitialState + tickGame + действия
    persistence.ts          ← сохранение в localStorage (версия схемы)
    speech.ts               ← Web Speech адаптеры (SpeechInput/OutputProvider)
    rng.ts, ids.ts, util.ts, constants.ts
  state/GameContext.tsx     ← React-биндинг: цикл времени, автосейв, озвучка
  components/               ← слой интерфейса (панели, карта, окна, обучение)
```

Правильная последовательность одного приказа:

```
текст/голос → CommandInterpreter.parse → извлечение исполнителя/цели/войск
→ проверка выполнимости → (если неполно) уточняющий вопрос офицера
→ формальный объект Order → симуляция исполняет → последствия
→ DialogueProvider формирует доклад → (опц.) озвучка
```

## Как расширять

### Добавить нового офицера
Добавьте запись в `scenario.NIGHT_SIEGE.officers` (`src/game/scenario.ts`):
имя, должность, `crestSeed`, `accentColor`, `traits` (0–100), `speechStyle`
(`stoic` / `brash` / `analytic`), стартовую локацию и войска. Портрет-герб
генерируется процедурно из `crestSeed`. Реплики подберутся по `speechStyle`.

### Добавить новую локацию
Добавьте объект в `scenario.locations`: координаты `x/y` (в % карты), `type`,
`effects` (оборона, бонус лучников, узкий проход, укрытие, обзор) и `roads`
(двусторонние — не забудьте обратную связь). Иконка берётся из `type`.

### Добавить новый тип приказа
1. Добавьте значение в `OrderAction` (`src/game/types.ts`).
2. Добавьте ключевые слова в `ACTION_KEYWORDS` (`commandInterpreter.ts`) и
   метку в `ACTION_LABELS` (`constants.ts`).
3. Обработайте исполнение в `applyArrival`/`startOrderExecution` (`engine.ts`).
4. При необходимости отметьте действие как опасное в `DANGEROUS_ACTIONS`.

### Заменить локальный разбор приказов на OpenAI
`CommandInterpreter` — это интерфейс (`src/game/types.ts`). Реализуйте
`OpenAICommandInterpreter` с методом `parse(text, context): ParsedCommand`
(например, function-calling, возвращающий ту же структуру) и подставьте его
вместо `localInterpreter` в `engine.submitCommand`. Игровая математика не
изменится — контракт `ParsedCommand` тот же.

### Заменить SpeechSynthesis на ElevenLabs
`SpeechOutputProvider` — тоже интерфейс. Реализуйте `speak(text, lang)` через
ElevenLabs (стриминг аудио) и подставьте вместо `speechOutput` в
`GameContext`. Аналогично `SpeechInputProvider` можно заменить на Whisper.

## Известные ограничения первой версии

- Один сценарий, пять офицеров, три типа войск — намеренно узкий срез.
- Разбор языка — эвристический (не LLM): понимает типовые формулировки, но не
  произвольный текст. Незнакомые фразы вызывают просьбу переформулировать.
- Вражеский ИИ — сценарный (движется по маршруту, набег на деревню, реакция на
  засаду), не обучаемый.
- «Фланговый удар» инициативы упрощён: конница присоединяется к бою у цели.
- Несовместимые старые сохранения отбрасываются (без миграций) — игра не падает.
- Голос зависит от браузера; текст всегда доступен как полноценная замена.

Точки расширения (несколько королевств, экономика, дипломатия, десятки
офицеров, внешние LLM/голос) заложены в интерфейсах и конфигурации сценария.
