# Growth-исследование проекта «Добро» — промежуточный документ

Источник: `concept-tz-draft.md` (раздел 4.8 «Growth-контур / вирусное
распространение»). Этот документ — самодостаточная выжимка контекста для
исследовательских агентов + план исследований + ТЗ на каждого агента.

---

## 1. Контекст проекта (кратко)

«Добро» — голосовой ИИ-компаньон в Telegram для аудитории 50+/пенсионеров.
Общается голосом, сам проявляет инициативу (чек-ины, напоминания, поздравления),
помогает по здоровью и быту (рецепты, растения, документы). Одна из ключевых
целей проекта — **вирусное распространение** генерируемого контента за пределы
пилотной группы, чтобы получить максимальную пользовательскую аудиторию.

**Гипотеза роста**: пожилой пользователь бота генерирует что-то тёплое/личное
(открытку, песню, стих) и отправляет в семейный чат (детям, внукам). Получатели
видят ценность и заводят такого же бота для своих родителей/бабушек-дедушек.

Уже решено по механике (см. концепт, п.4.8):
- Контент несёт ненавязчивую ссылку-приглашение «сделать такое же для своих».
- Дополнительно тестируем активный реферал (бот сам предлагает пригласить родных).
- На старте предпочитаем готовые бесплатные изображения (Pinterest-подобные
  источники) вместо ИИ-генерации — генерация как запасной вариант.
- Доставка в семейный чат — на выбор пользователя: вручную (forward) или бот
  сам публикует, если его добавили в чат.

**Не решено и требует исследования** — какие именно форматы контента дадут
максимальный виральный эффект. Кандидаты, названные в штурме:
- Персональные открытки: «доброе утро», с праздником, с днём рождения.
- Персональные песни (поздравления, посвящения).
- Персональные стихи/поздравления текстом.
- (Открыто для агентов: любые другие форматы, которые всплывут в исследовании —
  видео-поздравления, фотоколлажи, голосовые открытки и т.п.)

---

## 2. Цель исследования

Определить 2–3 топовых формата ИИ-контента для вирусного распространения в
семейных чатах (Telegram и аналоги) среди аудитории 50+/пенсионеров и их семей —
с опорой на реальные тренды, примеры и (где возможно) метрики вовлечённости —
чтобы дальше сфокусировать разработку MVP на самых перспективных направлениях,
а не распылять ресурсы на все форматы сразу.

---

## 3. План исследований (4 параллельных потока)

- **Поток A — AI-открытки/поздравительные карточки.** Что уже вирусит в формате
  персонализированных открыток (доброе утро, праздники, дни рождения) в
  русскоязычных и мировых мессенджер-аудиториях, особенно 50+/семейных чатах.
- **Поток B — AI-генерация персональных песен.** Сервисы генерации песен как
  подарка/поздравления (например, Suno и аналоги), примеры вирусных кейсов,
  восприятие такого контента старшей аудиторией и её детьми/внуками.
- **Поток C — AI-генерация персональных стихов/поздравлений текстом.** Тренды
  генеративных текстовых поздравлений, примеры ботов/сервисов, вовлечённость.
- **Поток D — Виральные механики и конкуренты.** Как устроены механики роста
  (K-фактор, рефералки) у похожих продуктов — ИИ-компаньоны для пожилых
  (Replika и аналоги, российские сервисы для пенсионеров/eldercare AI apps) и
  ботов с шаринг-контентом в семейные чаты.

## 4. ТЗ для агентов-исследователей

Общие требования к каждому агенту:
- Искать актуальную информацию в интернете (WebSearch/WebFetch), не полагаться
  только на память.
- Фокус на применимости к русскоязычной аудитории 50+ и формату Telegram/семейных
  чатов, но не игнорировать релевантные мировые примеры.
- В конце — короткая рекомендация: развивать этот формат в MVP / протестировать
  на пилоте / отложить — с обоснованием.
- Указывать источники (ссылки).
- Объём ответа — сжатый структурированный отчёт, не сырой список ссылок.

### ТЗ — Агент A (открытки/карточки)
Исследовать рынок персонализированных ИИ-открыток и поздравительных карточек
(«доброе утро», праздники, дни рождения) как формата вирусного контента для
пересылки в семейные чаты. Что делает такую открытку "расшариваемой" (юмор,
трогательность, персонализация с именем/фото)? Есть ли примеры ботов/сервисов,
уже завирусившихся на этом в РФ или мире, особенно среди аудитории 50+ и их
семей? Какие технические решения (шаблоны, ИИ-генерация изображений с текстом)
используются? Дать рекомендацию для проекта «Добро».

### ТЗ — Агент B (персональные песни)
Исследовать формат ИИ-генерации персональных песен как подарка/поздравления
(сервисы вроде Suno, Udio и аналогов, боты поверх них). Насколько это виральный
формат — есть ли кейсы распространения именно в семейных/родственных чатах?
Насколько аудитория 50+ (и как получатели-дети/внуки) реагирует на такой
контент? Какие есть барьеры (время генерации, качество, стоимость, сложность
восприятия для пожилых на входе — но получатель, скорее, ребёнок/внук)? Дать
рекомендацию для проекта «Добро».

### ТЗ — Агент C (персональные стихи/поздравления текстом)
Исследовать формат генеративных текстовых поздравлений/стихов как вирусного
контента. Есть ли существующие боты/сервисы такого рода, насколько они
популярны, есть ли метрики вовлечённости/расшаривания? Насколько текстовый
формат (без картинки/аудио) вообще виралится в мессенджерах по сравнению с
визуальным/аудио-контентом? В каком виде текст обычно комбинируют с
изображением/открыткой для эффекта? Дать рекомендацию для проекта «Добро».

### ТЗ — Агент D (виральные механики и конкуренты)
Исследовать существующие ИИ-компаньоны/ассистенты для пожилых людей (Replika и
аналоги, специализированные eldercare AI-боты в России и мире) — как у них
устроен рост аудитории, есть ли встроенные механики вирусного распространения
через контент, отправляемый в чаты/соцсети, реферальные программы, K-факторы.
Какие практики роста через "передачу подарка/контента близким" уже
зарекомендовали себя в похожих нишах (не только eldercare — любые ИИ-боты с
генерацией персонального контента для пересылки)? Дать рекомендацию по
механике вирусного распространения для проекта «Добро».

---

## 5. Формат ответа

Каждый агент возвращает текстовый отчёт со структурой:
1. Ключевые находки (тренды, примеры, кейсы).
2. Метрики вовлечённости/вирусности, если найдены.
3. Применимость к аудитории 50+ и формату Telegram/семейных чатов в РФ.
4. Рекомендация: развивать / протестировать / отложить — с обоснованием.
5. Источники.
